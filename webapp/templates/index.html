How to Read This Map

What am I looking at?

This interactive map shows how a machine learning model organizes microscopy experiments based on the visual characteristics of the cells.

Each point represents one experimental well. Wells that appear close together contain cells with similar visual patterns according to the model, while wells farther apart have more different learned representations.

The map is a two-dimensional visualization of a much larger feature space. The goal is to help explore patterns in the model's learned understanding of cellular morphology.

What does each point represent?

Each point represents a single well from a Cell Painting experiment.

The model converts the microscopy images from each well into a numerical representation called an embedding — a compact "fingerprint" of the cell images that captures important visual features learned by the model.

In this visualization, each well is represented by a 384-dimensional embedding that has been projected into two dimensions for visualization.

Learn more: Introduction to embeddings (TensorFlow)

What does distance mean?

The distance between points represents similarity in the model's learned representation.

  • Nearby points → cells with similar visual characteristics according to the model.
  • Distant points → cells with more different visual characteristics.

This similarity reflects phenotypic similarity — similarity in observable cellular appearance — and should not be interpreted as proof that two compounds have the same biological mechanism.

For example, two compounds may appear close together because they cause similar morphological changes, even if they act through different biological pathways.

What do the colors represent?

Colors indicate annotated mechanisms of action (MoA) when available.

A mechanism of action describes the biological process affected by a compound (for example, disrupting DNA replication or affecting a specific cellular pathway).

Only the most frequent mechanisms are shown as individual colors to keep the map readable. Less common mechanisms are grouped as "Other," and some wells may not have an available annotation.

Importantly, the model was not trained using these labels. The colors are added afterward to evaluate whether biologically meaningful patterns emerge in the learned representation.

Why are the axes labeled UMAP 1 and UMAP 2?

The original embeddings contain 384 dimensions, which cannot be directly visualized. UMAP (Uniform Manifold Approximation and Projection) is a dimensionality reduction method that compresses high-dimensional data into two dimensions while attempting to preserve local relationships.

The UMAP axes themselves do not represent specific biological features. The meaningful information is:

  • which points are close together,
  • which groups form clusters,
  • and how samples relate to each other.

Learn more: UMAP documentation

How is similarity calculated?

Each well is represented as a numerical embedding. Similarity between wells is calculated using cosine similarity, which compares the orientation of two embedding vectors.

Higher similarity means the model considers the wells to have more similar learned visual characteristics.

This is a measure of similarity in the model's feature space — not a direct measurement of biological similarity.

What does the attention map show?

The attention map highlights which regions of the microscopy image most influenced the model's representation of that well. Brighter regions indicate areas where the model placed more emphasis when analyzing the image, while darker regions indicate areas that contributed less.

Attention maps provide insight into what visual patterns the model used, but they do not directly identify specific cellular structures or biological mechanisms. A region with high attention is not necessarily the most biologically important feature — it is simply an area that the model relied on when creating its learned representation.

About the Project

What is Cell Painting?

Cell Painting is a high-content microscopy assay that uses fluorescent stains to visualize multiple structures inside cells.

Cells are stained to highlight features including:

  • nucleus,
  • mitochondria,
  • endoplasmic reticulum,
  • RNA,
  • cytoskeletal and membrane structures.

By capturing many cellular features at once, Cell Painting provides a detailed view of how compounds change cell morphology.

Learn more: Broad Institute Cell Painting overview

Why analyze cell morphology?

Changes in a cell's appearance can reveal how a compound affects cellular processes.

Different perturbations may alter features such as:

  • cell shape,
  • organelle organization,
  • nuclear morphology,
  • texture,
  • and overall cellular state.

These observable characteristics are called phenotypes.

The goal of this project is to determine whether machine learning can discover meaningful cellular phenotypes directly from images.

What did the model learn?

This project uses a self-supervised vision model called DINO (self-Distillation with No Labels).

Traditional supervised learning requires examples with known labels, such as: "This image came from a compound affecting pathway X."

Self-supervised learning instead allows a model to learn patterns directly from unlabeled data. During training, DINO learns a representation of microscopy images by identifying consistent visual patterns across different views of the same data.

Learn more: DINO: Emerging Properties in Self-Supervised Vision Transformers (arXiv)

Why is this approach interesting?

Large microscopy datasets contain enormous amounts of information, but manually labeling every image with biological annotations is expensive and incomplete.

Self-supervised learning provides a way to discover patterns without requiring predefined labels.

If samples with similar biological effects cluster together after training, it suggests that the model has learned meaningful representations of cellular morphology.

Dataset

This project uses data from the JUMP Cell Painting Consortium, an open resource containing large-scale Cell Painting microscopy experiments.

The dataset contains images of cells treated with thousands of compounds across many experimental conditions.

Learn more: JUMP Cell Painting Consortium

Technical Details

Computational pipeline

The visualization was generated through the following pipeline:

Cell Painting microscopy images
↓
DINO Vision Transformer
↓
384-dimensional embeddings
↓
Embedding postprocessing
↓
Cosine similarity analysis
↓
UMAP dimensionality reduction
↓
Interactive visualization

Model architecture

The representation model is a Vision Transformer (ViT) trained using the DINO self-supervised learning framework.

A Vision Transformer divides images into smaller regions and learns relationships between those regions to build a representation of the image.

The model outputs a 384-dimensional embedding representing learned visual features.

Learn more: Vision Transformer paper (Google Research)

Embeddings and similarity

An embedding is a numerical representation produced by the model.

Instead of comparing images directly pixel-by-pixel, embeddings allow the model to compare images based on learned features. For this project:

  • each well is represented by a 384-dimensional vector,
  • embeddings summarize learned morphological patterns,
  • similarity is measured using cosine similarity.

Embedding processing

Before visualization, embeddings are processed to improve comparisons between samples. Processing includes:

  • median absolute deviation (MAD) normalization,
  • sphering/whitening transformations,
  • normalization before similarity calculations.

These steps reduce unwanted variation and improve the structure of the learned feature space.

UMAP visualization

Because the embeddings exist in a 384-dimensional space, they cannot be directly displayed. UMAP creates a two-dimensional representation that preserves local relationships between samples as much as possible.

The final map is therefore an interactive window into the model's learned feature space rather than a direct biological coordinate system.
